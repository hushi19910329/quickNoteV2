# 快捷笔记软件数据库设计（SQLite）

## 1. 设计目标
- 支持便签富文本、图片引用、标签、多提醒与设置项持久化。
- 优先本地可靠性与查询性能，避免过度复杂。
- 使用软删除策略，保留恢复能力。

## 2. ER 关系概览
- `notes` 1 - N `attachments`
- `notes` 1 - N `reminders`
- `notes` N - N `tags`（通过 `note_tags`）
- `settings` 独立键值配置

## 3. 表结构

## 3.1 notes（便签主表）
| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PK AUTOINCREMENT | 便签ID |
| title | TEXT | NOT NULL DEFAULT '' | 标题 |
| content_html | TEXT | NOT NULL DEFAULT '' | 富文本内容 |
| plain_text | TEXT | NOT NULL DEFAULT '' | 纯文本索引内容 |
| color | TEXT | NOT NULL DEFAULT 'yellow' | 便签颜色 |
| emoji_icon | TEXT | NOT NULL DEFAULT '📝' | 便签图标 |
| is_pinned | INTEGER | NOT NULL DEFAULT 0 | 是否置顶便签 |
| is_archived | INTEGER | NOT NULL DEFAULT 0 | 是否归档 |
| is_deleted | INTEGER | NOT NULL DEFAULT 0 | 是否软删除 |
| created_at | TEXT | NOT NULL | 创建时间（ISO8601） |
| updated_at | TEXT | NOT NULL | 更新时间（ISO8601） |
| deleted_at | TEXT | NULL | 软删除时间 |

索引：
- `idx_notes_updated_at(updated_at DESC)`
- `idx_notes_is_archived(is_archived, is_deleted)`
- `idx_notes_color(color)`

## 3.2 tags（标签表）
| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PK AUTOINCREMENT | 标签ID |
| name | TEXT | UNIQUE NOT NULL | 标签名 |
| color | TEXT | NOT NULL DEFAULT '#4A90E2' | 标签颜色 |
| sort_order | INTEGER | NOT NULL DEFAULT 0 | 排序 |
| created_at | TEXT | NOT NULL | 创建时间 |

索引：
- `idx_tags_sort(sort_order, id)`

## 3.3 note_tags（便签-标签关联表）
| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| note_id | INTEGER | NOT NULL FK -> notes(id) | 便签ID |
| tag_id | INTEGER | NOT NULL FK -> tags(id) | 标签ID |
| created_at | TEXT | NOT NULL | 创建时间 |

主键与索引：
- 复合主键：`PRIMARY KEY(note_id, tag_id)`
- `idx_note_tags_tag_id(tag_id, note_id)`

## 3.4 attachments（附件表，主要用于图片）
| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PK AUTOINCREMENT | 附件ID |
| note_id | INTEGER | NOT NULL FK -> notes(id) | 所属便签 |
| file_path | TEXT | NOT NULL | 本地路径 |
| mime_type | TEXT | NOT NULL | 类型（如 image/png） |
| width | INTEGER | NULL | 展示宽度 |
| height | INTEGER | NULL | 展示高度 |
| created_at | TEXT | NOT NULL | 创建时间 |

索引：
- `idx_attachments_note(note_id, id)`

## 3.5 reminders（提醒表）
| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INTEGER | PK AUTOINCREMENT | 提醒ID |
| note_id | INTEGER | NOT NULL FK -> notes(id) | 关联便签 |
| remind_at | TEXT | NOT NULL | 下次提醒时间 |
| repeat_rule | TEXT | NULL | 重复规则（cron/rrule/json） |
| is_enabled | INTEGER | NOT NULL DEFAULT 1 | 是否启用 |
| snooze_minutes | INTEGER | NOT NULL DEFAULT 0 | 暂缓分钟数 |
| last_triggered_at | TEXT | NULL | 上次触发时间 |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

索引：
- `idx_reminders_due(is_enabled, remind_at)`
- `idx_reminders_note(note_id, is_enabled)`

## 3.6 settings（配置表）
| 字段名 | 类型 | 约束 | 说明 |
|---|---|---|---|
| key | TEXT | PK | 配置键 |
| value | TEXT | NOT NULL | 配置值（JSON字符串） |
| updated_at | TEXT | NOT NULL | 更新时间 |

典型键：
- `ui.opacity`
- `ui.always_on_top`
- `ui.collapsed_mode`
- `speech.base_url`
- `speech.api_key`
- `speech.model`
- `app.auto_start`

## 4. 初始化 SQL（建议）
```sql
PRAGMA foreign_keys = ON;

CREATE TABLE IF NOT EXISTS notes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL DEFAULT '',
  content_html TEXT NOT NULL DEFAULT '',
  plain_text TEXT NOT NULL DEFAULT '',
  color TEXT NOT NULL DEFAULT 'yellow',
  emoji_icon TEXT NOT NULL DEFAULT '📝',
  is_pinned INTEGER NOT NULL DEFAULT 0,
  is_archived INTEGER NOT NULL DEFAULT 0,
  is_deleted INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  deleted_at TEXT
);

CREATE TABLE IF NOT EXISTS tags (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  color TEXT NOT NULL DEFAULT '#4A90E2',
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS note_tags (
  note_id INTEGER NOT NULL,
  tag_id INTEGER NOT NULL,
  created_at TEXT NOT NULL,
  PRIMARY KEY (note_id, tag_id),
  FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS attachments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  note_id INTEGER NOT NULL,
  file_path TEXT NOT NULL,
  mime_type TEXT NOT NULL,
  width INTEGER,
  height INTEGER,
  created_at TEXT NOT NULL,
  FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS reminders (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  note_id INTEGER NOT NULL,
  remind_at TEXT NOT NULL,
  repeat_rule TEXT,
  is_enabled INTEGER NOT NULL DEFAULT 1,
  snooze_minutes INTEGER NOT NULL DEFAULT 0,
  last_triggered_at TEXT,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL,
  FOREIGN KEY (note_id) REFERENCES notes(id) ON DELETE CASCADE
);

CREATE TABLE IF NOT EXISTS settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_notes_updated_at ON notes(updated_at DESC);
CREATE INDEX IF NOT EXISTS idx_notes_is_archived ON notes(is_archived, is_deleted);
CREATE INDEX IF NOT EXISTS idx_notes_color ON notes(color);
CREATE INDEX IF NOT EXISTS idx_tags_sort ON tags(sort_order, id);
CREATE INDEX IF NOT EXISTS idx_note_tags_tag_id ON note_tags(tag_id, note_id);
CREATE INDEX IF NOT EXISTS idx_attachments_note ON attachments(note_id, id);
CREATE INDEX IF NOT EXISTS idx_reminders_due ON reminders(is_enabled, remind_at);
CREATE INDEX IF NOT EXISTS idx_reminders_note ON reminders(note_id, is_enabled);
```

## 5. 数据一致性与迁移策略
- 数据库版本控制：`PRAGMA user_version`。
- 每次升级执行增量迁移脚本（`migrations/v{n}.sql`）。
- 迁移必须在事务内执行，失败回滚。
- 对 `notes` 等核心表采用“新增字段 + 默认值 + 兼容读取”策略，避免破坏旧数据。

## 6. 安全与性能建议
- API Key 存储建议：优先系统凭据（如 Windows Credential Manager），数据库仅保存引用键。
- 搜索优化：首期使用 `plain_text LIKE`，后续可升级 FTS5。
- 大图策略：粘贴后生成压缩副本，正文只引用文件路径，避免 DB 过大。
