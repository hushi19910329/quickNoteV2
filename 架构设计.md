# 快捷笔记软件架构设计

## 1. 技术选型
- 语言：Python 3.11+
- UI：PySide6
- 数据库：SQLite（`sqlite3`）
- 富文本：`QTextEdit/QTextDocument`
- 提醒调度：`APScheduler`
- 通知：`plyer`（跨平台通知，Windows 可用）
- 语音转写：硅基流动 API（`requests`）

## 3. 分层职责
- **UI 层 (`ui/`)**
  - 负责界面展示、用户输入事件、视觉反馈。
  - 不直接操作数据库，通过 Controller/Service 调用业务。
- **Controller 层 (`controllers/`)**
  - 将 UI 事件转换成业务操作。
  - 组织 Service 调用并回填 UI 状态。
- **Service 层 (`services/`)**
  - 核心业务逻辑：笔记编辑、提醒调度、搜索、语音转写、窗口行为。
- **Repository 层 (`db/repositories/`)**
  - SQL 访问封装，提供稳定的数据读写接口。
- **Model 层 (`models/`)**
  - 数据实体定义（dataclass 或轻量对象）。

## 5. 关键流程设计

## 5.1 新建并编辑便签流程
1. UI 点击“新建”。
2. `note_controller.on_create_note()` 调用 `note_service.create_quick_note()`。
3. Repo 写入数据库并返回 `note_id`。
4. UI 自动选中并进入编辑态。
5. 编辑内容变化触发节流保存（800ms）到 `on_note_content_changed()`。

## 5.2 提醒触发流程
1. 用户设置提醒，`reminder_controller` 调用 `reminder_service.schedule_reminder()`。
2. 服务层写库并注册 APScheduler 任务。
3. 到点触发 `handle_trigger()`。
4. 弹出系统通知并刷新便签状态。

## 5.3 语音转写流程
1. 用户在 `VoiceInputDialog` 完成录音并提交。
2. `speech_service` 上传音频到硅基流动接口。
3. 转写结果返回后，由 UI 选择“插入到光标”或“追加到末尾”。
4. 自动保存到数据库。

## 6. 非功能要求
- 启动耗时目标：冷启动 < 2 秒（1000 条便签内）。
- 编辑保存可靠性：异常退出后最大内容丢失 < 2 秒。
- 提醒误差：本地触发误差 <= 2 秒。
- 可维护性：模块边界清晰，Service/Repo 可单测。
